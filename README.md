# 周末去哪儿抽签

一个给 25-35 岁中国年轻情侣使用的周末/小长假目的地抽签网页。目的地库同时包含地级市、县城、县级市和区县型度假目的地；打开 `index.html` 即可使用，不依赖外部服务。

## 使用说明

1. 搜索并选择出发城市。出发城市已覆盖全国地级市、自治州、地区、盟、直辖市和港澳台入口。
2. 选择交通范围和出行时长。
3. 在“已经去过”里点亮目的地，它们会直接从抽签候选里排除。
4. 点击“开始抽签”，网页会同时给出一个常规答案和一个惊喜答案，附带推荐理由、交通估算和路线。
5. 两个答案都可以选，常规答案更贴近交通范围，惊喜答案保留一点反直觉可能性。

## 设计口径

- 自驾范围分为 1～2 小时和 2～4 小时两档，使用城市经纬度粗略估算，适合筛选，不替代地图导航。
- 如果某个出发地周边目的地库暂时偏少，系统会补入最近的区域备选，避免只抽到本城。
- 目的地库包含精选旅行地和城市探索型目的地，后者强调地方馆、生活街区、市场小吃、河湖海边或城市公园等真实周末玩法。
- 4 小时高铁圈优先使用常见城市间高铁/城际时间口径，缺失线路用距离做保守估算。
- 目的地亮点参考国家级旅游休闲街区、国家级夜间文化和旅游消费集聚区、典型城市/县域景区、地方文旅资料和公开地理资料。
- 县域目的地已覆盖青田县、松阳县、云和县、嘉善县、常熟市、安吉县、德清县、桐庐县、淳安县、临海市等，适合“不是非去大城市不可”的周末决策。
- 抽签更强调随机性，不展示候选概率排行，不再要求用户设置实际偏好；已去过目的地会直接排除。
- 页面面向手机和 PC 自适配，所有主要文字和按钮都按触屏尺寸设计。

## 本地文件

- `index.html`：页面结构
- `styles.css`：视觉样式和响应式布局
- `app.js`：城市库、加权抽签、页面交互
- `data/departures.js`：全国出发城市数据
- `assets/weekend-map.png`：项目内生成的视觉背景
- `tests/logic.test.js`：核心逻辑测试
